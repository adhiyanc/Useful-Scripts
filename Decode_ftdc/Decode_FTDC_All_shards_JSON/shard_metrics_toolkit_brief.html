Interactive brief • shard metrics toolkit

One script orchestrates the collection. One helper decodes FTDC on each host.

Together these scripts collect FTDC-derived opcounter deltas across many shards, move the work to the remote host over SSH, and export both raw JSON and flattened CSV for analysis.

Script A

Orchestrator

Reads PRIMARY_TARGETS, validates the requested window, streams the decoder over SSH, and writes per-shard plus combined outputs.

Script B

Decode helper

Runs on a single host, parses MongoDB FTDC metrics files, reconstructs values from delta-encoded chunks, and returns first/last/delta for selected opcounters.

End-to-end workflow

This is the combined operator flow across both scripts.

1

Set targets

The orchestrator reads shard and host pairs from PRIMARY_TARGETS.

2

Pass time window

You supply start, end, and dir-prefix to the orchestrator. It validates them before any remote work starts.

3

Stream helper

The orchestrator sends the decode helper to each host over SSH and runs python3 - with shard, directory, start, and end arguments.

4

Decode FTDC

The helper selects relevant metrics.* files, decompresses chunks, reconstructs sample series, and extracts six opcounter metrics inside the requested window.

5

Aggregate outputs

The orchestrator saves per-shard JSON, writes all_shards.json, and flattens the metric summaries into all_shards.csv.

Inputs and outputs

The two scripts share a clean handoff: the orchestrator passes arguments, the helper returns JSON.

Inputs

Operator-facing inputs:

  • PRIMARY_TARGETS: Python-style list of (shard, host) pairs
  • --start and --end: inclusive time window
  • --dir-prefix: absolute path prefix on each remote host
export PRIMARY_TARGETS="[('shard01','host1.example.com'),('shard02','host2.example.com')]" python3 collect_metrics.py \ --start 2026-07-07T10:00:00Z \ --end 2026-07-07T11:00:00Z \ --dir-prefix /d/d3/sem2/qa
Outputs

Main artifacts written by the orchestrator:

  • shard_outputs/<shard>.json
  • shard_outputs/all_shards.json
  • shard_outputs/all_shards.csv

The helper returns matched sample count, selected metrics files, and per-metric first/last/delta values for the target opcounters.

Where to debug

A quick decision tree for common failures.

The orchestrator fails before SSH
  • Check PRIMARY_TARGETS syntax
  • Check start and end format
  • Confirm dir-prefix is absolute
The helper returns no metrics files
  • Confirm the remote path points to diagnostic.data
  • Confirm files match metrics.<timestamp>Z-<n>
Remote stdout was not valid JSON
  • The helper should print only JSON on stdout
  • Inspect saved per-shard stdout/stderr in shard_outputs/<shard>.json
The CSV is empty
  • No samples matched the requested window
  • Or none of the six target opcounter metrics were present
Bottom line: treat the orchestrator as the fleet runner and the helper as the FTDC-aware metric extractor. If the helper JSON is good, the aggregate outputs should be straightforward.
