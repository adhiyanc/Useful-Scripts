📖 Overview

FTDC Analysis is a powerful component of the mongo-x-ray tool that analyzes MongoDB's Full Time Diagnostic Data (FTDC) captures to provide comprehensive performance insights.

What is FTDC?

FTDC is MongoDB's diagnostic data collection system that continuously captures performance metrics from your MongoDB instances. It includes:

  • Operation counters and latencies
  • Host system metrics (CPU, memory, I/O)
  • WiredTiger cache statistics
  • Disk and mount point utilization
  • Replication set member states

Key Features

📈 Baseline Analysis

Summarizes workload and performance metrics with peak and average values.

📊 Charts & Visualizations

Generates interactive charts for metrics including PNG and SVG formats.

⏱️ Time Range Selection

Analyze specific time windows within your FTDC data.

🎯 Sampling Control

Fine-tune data sampling rates for large captures.

🚀 Quick Start

Basic Usage

Analyze FTDC data from your MongoDB diagnostic directory:

x-ray ftdc /var/lib/mongo/diagnostic.data

Analyze Specific Time Range

Focus on a specific timeframe using UTC ISO-8601 timestamps:

x-ray ftdc /var/lib/mongo/diagnostic.data \ 2026-06-17T08:00:00Z 2026-06-17T10:00:00Z

Generate Markdown Report

x-ray ftdc /var/lib/mongo/diagnostic.data \ --output ./reports/ \ --format markdown
✅ Success: Your analysis report will be saved in the specified output directory!

💻 Commands

Full Syntax

x-ray ftdc [-h] [-s CHECKSET] [-o OUTPUT] [-f {markdown,html}] [-r RATE] [--svg] ftdc_path [start_time] [end_time]

Arguments

Argument Type Default Description
ftdc_path Required — Path to directory containing FTDC files (metrics.*)
start_time Optional First data point Inclusive UTC start time (ISO-8601 format)
end_time Optional Last data point Inclusive UTC end time (ISO-8601 format)

Options

Option Short Default Description
--checkset -s default Checkset to run (from config)
--output -o output/ Output folder path
--format -f html Output format: markdown or html
--rate -r 1 / files Sampling rate (0 to 1)
--svg — false Use SVG format for charts instead of PNG
--help -h — Show help message

📝 Examples

Example 1: Basic Analysis with HTML Output

x-ray ftdc /var/lib/mongo/diagnostic.data \ --output ./ftdc_reports/

Result: Generates a complete HTML report in ./ftdc_reports/

Example 2: Analyze Last 24 Hours

x-ray ftdc /var/lib/mongo/diagnostic.data \ 2026-06-16T08:00:00Z 2026-06-17T08:00:00Z

Example 3: Custom Sampling Rate for Large Captures

x-ray ftdc /var/lib/mongo/diagnostic.data \ --rate 0.1 \ --output ./ftdc_reports/

Result: Analyzes 10% of the data for faster processing

Example 4: Generate SVG Charts

x-ray ftdc /var/lib/mongo/diagnostic.data \ --svg \ --format html

Example 5: Using Custom Checkset

x-ray ftdc /var/lib/mongo/diagnostic.data \ --checkset performance_focused \ --format markdown

⚙️ Configuration

Configuration File Location

By default, x-ray uses the built-in configuration at: libs/config.json

You can specify a custom config file:

x-ray ftdc /var/lib/mongo/diagnostic.data \ --config ./custom_config.json

Environment Variables

Variable Values Effect
ENV development, production In development mode: formatted JSON output and same output folder reuse
LOG_LEVEL DEBUG, INFO, ERROR Controls verbosity of logging output

Sample Configuration Structure

The FTDC configuration includes:

{ "ftdc": { "template": "ftdc/full.html", "ftdcsets": { "default": { "items": [ "BaselineAnalysisItem", "MetadataReviewItem" ] } }, "item_config": { "BaselineAnalysisItem": { "sample_rate": 1.0, "max_sample_gap_seconds": 5 } } } }

Analysis Items

  • BaselineAnalysisItem: Generates baseline performance metrics and charts
  • MetadataReviewItem: Displays MongoDB configuration and system information

📊 Output

Report Structure

The FTDC Analysis generates a comprehensive report including:

  • report.html or report.md - Main analysis report
  • charts/ - Subdirectory containing all metric charts (PNG or SVG)
  • Metadata - MongoDB configuration, build info, host information

Report Sections

📈 Workload Metrics

Query, insert, update, delete operation rates

⏱️ Latency Analysis

Read/write/command operation latencies

💾 Performance Data

CPU, memory, disk I/O, and cache utilization

🖥️ Host Information

System metadata and resource limits

Metrics Included

Each metric report shows:

  • Peak Value: Maximum observed value during the time range
  • Average Value: Mean value over the time range
  • Warning Threshold: Configured threshold for alerting
  • Chart: Visual representation of metric over time
ℹ️ Note: All timestamps in the report are in UTC (Coordinated Universal Time).

🔧 Troubleshooting

Issue: "FTDC folder not found"

⚠️ Problem: The path doesn't exist or isn't a directory

Solution:

  • Verify the path is correct and points to the diagnostic.data directory
  • Check file permissions allow reading
  • Ensure the directory contains FTDC files (metrics.*)

Issue: "FTDC start time must be before or equal to end time"

⚠️ Problem: The start_time is after end_time

Solution:

  • Swap the timestamps to have start_time before end_time
  • Ensure timestamps are in ISO-8601 UTC format: YYYY-MM-DDTHH:MM:SSZ

Issue: "Config file not found"

⚠️ Problem: Custom config file is missing

Solution:

  • Verify the config file path is correct
  • Use the built-in config by omitting the --config option

Issue: "FTDC item failed" warnings

⚠️ Problem: One or more analysis items encountered errors

Solution:

  • Check the FTDC data files are not corrupted
  • Increase log level: set LOG_LEVEL=DEBUG
  • Verify the FTDC files contain the expected metrics

Performance Tips

  • Large Captures: Use --rate 0.1 to analyze 10% of data
  • Time Ranges: Specify start_time and end_time to focus analysis
  • Format: Markdown is faster than HTML generation
  • SVG vs PNG: SVG generation is faster but PNG is more compatible
✅ Tip: For very large FTDC captures (>1GB), use sampling and time range filtering together for optimal performance.
